# PRD: Página Web de Cuenta Regresiva (Salsa y Bachata)

Página web sencilla y de bajo mantenimiento que muestra información actualizada y una cuenta regresiva para todas las clases activas y pendientes de salsa y bachata, utilizando Google Sheets como base de datos.

## Quick path

1. **Fuente de verdad:** Google Sheets (publicado en la web para lectura pública).
2. **Hosting:** GitHub Pages (gratuito).
3. **Frontend:** HTML5, CSS3, Vanilla JavaScript (sin frameworks).
4. **Filtrado:** El frontend filtrará y ocultará automáticamente las clases cuya ventana de tiempo ya haya cerrado (FechaHora + DuracionMinutos).
5. **Cache:** Las peticiones al Sheet se almacenan en memoria por 15 segundos para evitar fetch innecesarios.

## Detalles

| Requisito | Decisión |
|-------|----------|
| **Carga de Datos** | Fetch directo al CSV/JSON de Google Sheets publicado para evitar API Keys o backend. |
| **Visualización** | Se mostrarán **todas las clases activas y pendientes** (sin límite de cantidad, ordenadas por fecha). |
| **Filtrado de vencidas** | Una clase se oculta cuando `now > FechaHora + DuracionMinutos`. Si `now` está dentro de esa ventana, se muestra el estado "¡Ahora!". |
| **Cuenta Regresiva** | Contador en tiempo real (días, horas, minutos, segundos) antes de que inicie la clase. |
| **Estado "En curso"** | Mientras `FechaHora <= now <= FechaHora + DuracionMinutos` se muestra "¡Clase ahora!" o "En vivo". |
| **Cache** | Los datos del Sheet se cachean en memoria por **15 segundos**. Peticiones dentro de ese período devuelven los datos ya cargados sin nuevo fetch. |
| **Diseño / UI** | Fondo oscuro con acentos vibrantes (rojo, dorado, magenta). Diseño responsivo (móvil y desktop). |

## Modelo de Datos (Google Sheets)

Columnas requeridas:

| Columna | Tipo | Requerida | Descripción |
|---------|------|-----------|-------------|
| `Clase` | Texto | ✅ | Salsa o Bachata |
| `FechaHora` | Texto (YYYY-MM-DD HH:mm) | ✅ | Inicio de la clase. Base para el countdown y el estado "Ahora". |
| `DuracionMinutos` | Número entero | ✅ | Duración en minutos. Define hasta cuándo mostrar el estado "En vivo". |
| `DiaSemana` | Texto | ➖ | Etiqueta de presentación (ej. "Martes"). |
| `Descripcion` | Texto | ➖ | Descripción libre de la clase. |
| `Link` | URL | ➖ | URL de Zoom, ubicación, etc. |
| `Activa` | Sí/No | ✅ | Si es "No", la clase se ignora sin importar la fecha. |

## Lógica de Estado de una Clase

```
now < FechaHora                                → Mostrar countdown
FechaHora <= now <= FechaHora + DuracionMinutos → Mostrar "¡Clase ahora!" / "En vivo"
now > FechaHora + DuracionMinutos              → Ocultar (clase vencida)
Activa = No                                    → Ocultar siempre
```

## Fuera de Alcance

- Sistemas de inscripciones y pagos.
- Login de usuarios o panel de administración web.
- Versión móvil nativa (app).

## Checklist de Aceptación

- [ ] La página renderiza sin errores en navegadores de PC y móviles.
- [ ] Solo se muestran clases cuya ventana `[FechaHora, FechaHora + DuracionMinutos]` esté activa o sea futura.
- [ ] Los contadores disminuyen en tiempo real y cambian a estado "En vivo" al llegar a cero.
- [ ] Al terminar `DuracionMinutos`, la card desaparece sin recargar la página.
- [ ] Dos fetches consecutivos dentro de 15 segundos no generan una segunda petición al Sheet.
- [ ] La web refleja los cambios del Sheet al refrescar (o al vencer la cache de 15 segundos).
